## Findings considered and rejected

- Local agent `spawn` during repair handoff: accepted product behavior because the PRD requires launching a selected local agent after explicit confirmation.
- Writing `.skills-doctor/reports`: accepted product behavior for repair handoff artifacts.
- Missing full semantic trigger evals: accepted MVP non-goal; the PRD explicitly excludes expensive trigger/output eval suites from MVP.
- Re-reporting plans 001-011: rejected because `plans/README.md` marks them DONE and recent commits show the findings were implemented.
- Human summary does not print diagnostics: considered but not planned in this pass because plan 002 already established diagnostics in reports and exit-code decisions; this would be a follow-up UX polish, not a fresh high-leverage correctness fix.
- Re-opening missing-skill child-directory behavior: rejected because plan 013 is DONE and `test/domain-scan.test.ts` now encodes ignoring non-skill child directories.
- Re-opening custom-root discovery support: rejected because plan 015 is DONE and `test/cli-scan.test.ts` covers adding a custom skills path while standard roots exist.
- Missing CI: rejected because `.github/workflows/ci.yml` exists and runs `bun run verify` on pull requests and pushes to `main`.
- Local agent `claude`/`codex` approval-bypass flags: accepted product behavior because `docs/PRD.md` requires local repair-agent launch only after explicit confirmation.
- Re-opening missing full trigger/output eval execution: rejected because the PRD names expensive eval suites as an MVP non-goal, and the scanner already reports missing `evals/evals.json` for non-trivial skills.
- Directly implementing the 2026-06-18 audit findings in the advisor pass: rejected by the improve skill workflow; source edits belong to executor agents following plans 024-039.
